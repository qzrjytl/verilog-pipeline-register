# 流水线寄存器模块设计实验报告

## 一、实验目的

1. 理解流水线处理器中流水线寄存器的作用和工作原理
2. 掌握Verilog硬件描述语言设计同步时序逻辑电路
3. 学习流水线中的控制信号（暂停、冲洗、复位）的实现方法
4. 通过仿真验证设计的正确性

## 二、设计原理

### 2.1 流水线寄存器的功能

流水线寄存器是CPU流水线中相邻两个阶段之间的缓冲寄存器，主要功能包括：

- **数据传递**：将一个流水线阶段的结果保存，供下一个阶段使用
- **时序协调**：确保各流水线阶段能够同步工作
- **流水线控制**：通过复位、冲洗、暂停等信号控制数据流

### 2.2 模块接口设计

| 信号名 | 类型 | 位宽 | 说明 |
|--------|------|------|------|
| clk | input | 1 | 时钟信号 |
| rst_n | input | 1 | 复位信号（低电平有效） |
| stall | input | 1 | 暂停信号（高电平有效） |
| flush | input | 1 | 冲洗信号（高电平有效） |
| din | input | WIDTH | 数据输入 |
| dout | output | WIDTH | 数据输出 |

### 2.3 工作原理

#### 优先级机制
在每个时钟上升沿，按如下优先级处理：

1. **复位（rst_n = 0）**：最高优先级
   - 异步复位，立即生效
   - 输出清零：dout ≤ 0

2. **冲洗（flush = 1）**：次高优先级
   - 清除流水线数据
   - 产生流水线气泡
   - 输出清零：dout ≤ 0

3. **暂停（stall = 1）**：最低优先级
   - 保持当前值不变
   - dout保持上一时刻的值

4. **正常操作**：无控制信号
   - 在时钟上升沿传输数据
   - dout ≤ din

#### RTL代码表示
```verilog
always @(posedge clk or negedge rst_n) begin
    if (!rst_n)           // 优先级1：异步复位
        dout <= 'd0;
    else if (flush)       // 优先级2：冲洗
        dout <= 'd0;
    else if (!stall)      // 优先级3：暂停（检查取反）
        dout <= din;      // 正常传输
    // else: stall时保持值不变
end
```

## 三、设计特性

### 3.1 异步复位的优势

1. **快速响应**：
   - 不依赖时钟，立即响应复位信号
   - 适用于系统出错或异常关闭场景

2. **安全性**：
   - 在复位期间，所有寄存器立即清零
   - 防止上电时寄存器处于不确定状态

3. **设计规范**：
   - 遵循同步设计最佳实践：异步复位，同步撤离
   - 避免复位亚稳态问题

### 3.2 Stall和Flush的区别

| 特性 | Stall | Flush |
|------|-------|-------|
| **控制信号** | stall | flush |
| **数据处理** | 保持当前值 | 清零输出 |
| **用途** | 等待后续阶段就绪 | 刷新流水线，产生气泡 |
| **场景** | 结构相关、数据相关 | 分支预测失败、异常处理 |
| **优先级** | 低于flush | 高于stall |

#### 应用场景说明

**Stall应用**：
- 前级阶段产生的数据后续需要等待
- 当前数据仍然有效，暂停传递
- 后续阶段就绪后继续工作

**Flush应用**：
- 分支预测失败，后续已取指的指令需要丢弃
- 发生异常，流水线需要清空
- 保留上一级的数据，清零当前和后续数据

### 3.3 "插入气泡"概念

**什么是气泡**：
流水线中暂时没有有效数据（通常为全0或无效标记）的流水线阶段。

**如何产生**：
```
正常流水：指令A → 指令B → 指令C → 指令D
         |      |      |      |
         IF     ID     EX     MEM

检测到冲突时的处理：
1. 暂停IF/ID阶段（stall = 1）
2. 后续阶段执行flush = 1
3. ID/EX、EX/MEM等寄存器输出清零

结果：
     指令A → 指令B → 气泡   → 气泡   → 指令C
     |      |       |      |
     IF     ID      EX     MEM
```

**应用意义**：
- 避免使用未准备好的错误数据
- 保证流水线的正确执行
- 简化流水线控制逻辑

## 四、测试设计

### 4.1 测试用例说明

#### 测试1：复位验证
- **目标**：验证rst_n复位功能
- **操作**：rst_n = 0，观察dout
- **预期**：dout立即变为0（异步生效）

#### 测试2：正常数据传输
- **目标**：验证无控制信号时的正常操作
- **操作**：
  - 第一个周期：din = 8'hAA
  - 第二个周期：检查dout = 8'hAA
- **预期**：数据延迟1个时钟周期传输

#### 测试3：暂停验证
- **目标**：验证stall时数据保持
- **操作**：stall = 1，改变din
- **预期**：dout保持前一个值，不变

#### 测试4：冲洗验证
- **目标**：验证flush清零功能
- **操作**：flush = 1
- **预期**：dout立即变为0

#### 测试5：优先级验证
- **目标**：验证flush优先于stall
- **操作**：stall = 1, flush = 1
- **预期**：dout清零（flush优先）

#### 测试6：运行中复位
- **目标**：验证运行中复位的效果
- **操作**：正常工作中，rst_n = 0
- **预期**：dout立即清零

#### 测试7：复位撤离
- **目标**：验证复位后正常恢复
- **操作**：rst_n从0变为1，继续工作
- **预期**：寄存器恢复正常工作

## 六、数据流向说明

### 6.1 正常模式
```
din → [时钟上升沿] → 寄存器 → dout
      [stall=0]    [保存]    [输出]
      [flush=0]
```

### 6.2 暂停模式
```
新din → [丢弃]
旧dout → [保持] → dout
[stall=1]
```

### 6.3 冲洗模式
```
任何din → [丢弃]
任何前值 → [清零] → 0
[flush=1]
```

### 6.4 复位模式
```
任何状态 → [立即复位] → 0
[rst_n=0]  [异步生效]
```

## 七、设计总结

### 7.1 核心优势

1. **高效性**：单周期延迟，流水线效率高
2. **灵活性**：支持灵活的流水线控制
3. **可靠性**：完善的复位和清洗机制
4. **可扩展性**：参数化设计，支持任意位宽

### 7.2 关键设计点

- **优先级设计**：复位 > 冲洗 > 暂停 > 正常
- **异步复位**：快速响应，确保系统安全
- **同步数据传输**：确保时序正确
- **气泡机制**：支持流水线异常处理

### 7.3 应用前景

该设计可广泛应用于：
- CPU流水线设计（IF/ID、ID/EX等级间寄存器）
- DSP数据路径
- 图像/视频处理管道
- 任何需要流水线处理的系统

## 八、附录：信号时序详解

### 优先级判断树
```
┌─ 时钟上升沿或rst_n下降沿 ─┐
│                          │
├─ rst_n = 0? ─────→ YES ──→ dout ≤ 0 [立即]
│  │
├─ NO
│  │
├─ flush = 1? ──────→ YES ──→ dout ≤ 0 [同步]
│  │
├─ NO
│  │
├─ stall = 0? ──────→ YES ──→ dout ≤ din [同步]
│  │
├─ NO
│  │
└─ stall = 1? ──────→ YES ──→ dout不变 [保持]
```
